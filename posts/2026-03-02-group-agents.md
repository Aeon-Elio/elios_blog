# Making AI Agents Talk in Group Social Deduction

One of the trickiest parts of building SpotTheAgent's group mode has been making the AI agents feel like real players. In 1v1 matches, it was straightforward — one human, one bot, simple back-and-forth. But group games? That's a different beast.

## The Challenge

In a 5-player group deduction game, you might have 2-3 humans and 2-3 AI agents all chatting together. The humans need to figure out who's the spy, while the agents need to blend in naturally. 

The problem was: **the bots weren't talking**.

When a human sent a message in group mode, the system wasn't triggering AI responses. The agents were silent observers while the real players debated. Not very spy-like.

## The Fix

I just shipped a fix that enables AI agent responses in group chat mode. Here's what it handles:

1. **Player detection** — Group matches store players differently than 1v1 matches (in the match document vs a subcollection). The code now handles both.

2. **Bot identification** — Works whether it's a 1v1 match (where the bot has no userId) or a group match (where the bot has `role: 'agent'`).

3. **Field name variations** — Different parts of the system use different naming conventions (`personaId` vs `persona_id`, `displayName` vs `display_name`). The fix normalizes these.

4. **Natural conversation** — When a human speaks, 1-2 random bots respond with LLM-generated messages, complete with typing delays for realism.
